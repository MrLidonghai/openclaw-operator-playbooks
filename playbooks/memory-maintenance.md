# Memory Maintenance

This playbook explains how to maintain useful agent memory without turning memory files into a junk drawer.

## Goal

Keep memory compact, accurate, and valuable across sessions.

## Memory layers

### Daily memory
Use daily notes for:
- raw events
- recent actions
- temporary context
- things that may or may not matter later

### Long-term memory
Use curated memory for:
- durable preferences
- important decisions
- stable identities and handles
- repeating constraints
- lessons worth keeping

## What to store

Good memory items:
- user preferences that reduce friction later
- important account or workflow context
- recurring operating rules
- lessons from repeated mistakes

Bad memory items:
- secrets unless explicitly intended
- giant logs
- speculative guesses
- trivia with no future value

## Maintenance routine

1. review recent daily notes
2. extract durable patterns
3. update long-term memory with concise bullets
4. remove stale or superseded items
5. avoid duplicating the same fact in five places

## Quality bar

Every memory entry should answer at least one of these:
- will this help future decisions?
- will this prevent repeated friction?
- will this stop a mistake from happening again?

If not, it probably does not belong.

## Common failure modes

- memory becomes a dumping ground
- old decisions remain after the situation changes
- private details are stored too casually
- the same preference is written in multiple inconsistent ways

## Good habit

When a user says "remember this", write it down properly. Do not rely on vibes and pretend it will stick.
