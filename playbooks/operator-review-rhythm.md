# Operator Review Rhythm

A good operator does not hover over the agent constantly.
A good operator also does not disappear for a week and hope nothing weird happened.

The answer is rhythm.

## Goal

Create a review cadence that keeps the system aligned without killing its momentum.

## Daily

Check:
- what the agent completed
- whether any failures repeated
- whether scheduled tasks behaved cleanly
- whether memory and improvement logs still make sense

You are not redoing all the work. You are scanning for drift.

## Weekly

Review:
- improvement trends
- false positives from automation
- noisy prompts or playbooks
- any creeping permission expansion
- what should be simplified or retired

Weekly review is where you prune.

## Monthly

Decide:
- which workflows are actually valuable
- which automations deserve stronger trust
- which repos, docs, or public outputs are worth refining
- whether the system is becoming more useful or just more busy

## Signals that need immediate review

- unexpected external actions
- repeated self-edit attempts in unstable areas
- larger diffs than usual
- vague justifications for changes
- schedules that create chatter without value

## Practical rule

Do not review everything equally.

Give attention to:
- external side effects
- risky changes
- repeated failures
- new automation surfaces

Give less attention to:
- stable low-risk routines that have already proved themselves

## Why rhythm matters

Without rhythm, operators either micromanage or abdicate.
Neither scales.

A calm, consistent review cadence is what lets an autonomous system remain genuinely supervised.
