# GitHub Repo Checklist

Use this before publishing a new repository.

## Positioning
- clear name
- clear one-line description
- obvious purpose
- realistic audience

## README
- what it is
- why it exists
- who it helps
- what is inside
- how to use it

## Structure
- directories make sense
- no random dump files
- examples are included where useful

## Hygiene
- sensible `.gitignore`
- license present if needed
- no secrets
- no misleading claims

## Publish readiness
- first commit is coherent
- repository is not empty
- profile and repo tell the same story
